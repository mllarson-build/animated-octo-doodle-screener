## Vision

### 10x Check
Instead of just a paper trading dashboard, build a **Personal Trading Intelligence System**: SQLite backbone, every screen run logged, signal source + thesis + conviction on every trade, portfolio correlation warnings, holding period accountability, and a "My Edge" analytics tab that teaches you about your own trading behavior. The system doesn't just track trades — it actively exposes your behavioral patterns and proves (or disproves) your signal edge.

### Platonic Ideal
You sit down at 8:30 AM. The tool shows: "3 new signals overnight. Value Recovery flagged XOM (score 6/7). Your historical win rate on Value Recovery signals with score 5+ is 64%. Entry zone: $108-110. Your portfolio is currently 40% tech — this would diversify into energy." The feeling: calm confidence backed by data.

## Implementation Approach

### Phase 1: SQLite Migration (~45 min with CC)

**Step 1: Create database schema (data/screener.db)**

```sql
CREATE TABLE IF NOT EXISTS trades (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp TEXT NOT NULL,
    ticker TEXT NOT NULL,
    entry REAL,
    target REAL,
    stop REAL,
    rr REAL,
    setup_score INTEGER,
    upside_pct REAL,
    shares INTEGER DEFAULT 0,
    summary TEXT DEFAULT '',
    actual_entry REAL,
    actual_exit REAL,
    outcome_notes TEXT DEFAULT '',
    status TEXT DEFAULT 'Watching',
    -- New columns
    signal_source TEXT DEFAULT 'Unknown',
    exit_reason TEXT DEFAULT '',
    thesis TEXT DEFAULT '',
    conviction INTEGER DEFAULT 0 CHECK(conviction BETWEEN 0 AND 5),  -- 0 = unscored/imported, 1-5 = user-rated. Filter conviction=0 from behavioral analytics.
    suggested_hold_period TEXT DEFAULT '',
    sector TEXT DEFAULT '',
    closed_at TEXT DEFAULT '',
    created_at TEXT DEFAULT (datetime('now'))
);

CREATE INDEX IF NOT EXISTS idx_trades_signal_source ON trades(signal_source);
CREATE INDEX IF NOT EXISTS idx_trades_status ON trades(status);
CREATE INDEX IF NOT EXISTS idx_trades_ticker ON trades(ticker);
```

Use WAL mode for safety (`PRAGMA journal_mode=WAL;`) even though single-user.

**Step 2: CSV import with backward compatibility**
- On first run, check if `data/trade_log.csv` exists AND `data/screener.db` does not
- Import all rows, filling missing columns with defaults (signal_source='Unknown', conviction=0, etc.)
- Skip malformed rows, log count: "Imported N trades, skipped M malformed rows"
- After successful import, rename CSV to `data/trade_log.csv.bak` (preserve backup)
- All subsequent operations use SQLite only

**Step 3: Update all data functions**
- `save_trade_idea` -> INSERT into trades table. Auto-populate `sector` from yfinance ticker info (already fetched by `analyze_trade`).
- `load_trade_log` -> SELECT from trades table, return DataFrame
- `save_trade_log` -> UPDATE trades table (for inline edits)
- `compute_paper_stats` -> extend with signal_source groupby queries
- `fetch_current_prices` -> unchanged (still uses yfinance)

### Phase 2: Core Features (~30 min with CC)

**Step 5: Paper trading dashboard**
Port the existing uncommitted diff (which adds compute_paper_stats, live P&L tracking, watchlist/open/closed sections in app.py) to use SQLite instead of CSV. The uncommitted code in app.py lines 843-994 and trade.py lines 417-557 contains this functionality.

**Step 6: Signal source attribution**
- Set `st.session_state["last_screen"] = "Value Recovery"` (etc.) when user loads data in each screen tab
- In Trade Builder, read `st.session_state.get("last_screen", "")` as default signal_source
- Show dropdown fallback: "Value Recovery", "Growth Momentum", "Options", "Manual"
- Existing status model preserved: Watching -> Executed -> Closed (Win/Loss) -> Cancelled

**Step 7: Close trade flow**
- "Close Trade" button per open position in the paper trading dashboard
- Default exit price = live price (with manual override input field)
- Exit reason dropdown: Hit Target / Hit Stop / Manual Exit / Thesis Invalidated
- Sets status to "Closed -- Win" or "Closed -- Loss" based on P&L sign
- Populates actual_exit, exit_reason, closed_at timestamp

### Phase 3: Expansion Features (~30 min with CC)

**Step 8: Thesis + conviction on save**
- Add text_area("Trade Thesis", placeholder="Why are you taking this trade?") to save flow
- Add slider("Conviction", 1, 5, 3) to save flow
- Both saved to trades table

**Step 9: Portfolio sector concentration warning**
- When saving a trade, query open Executed positions from trades table
- Group by sector, sum shares * entry price for dollar allocation
- Rule: warn if adding this trade would put >40% of total open position value in one sector
- Display: st.warning("Adding {ticker} would put {pct}% of your open portfolio in {sector}. You already have {N} positions there.")

**Step 10: Holding period countdown**
- In open positions display, show: "Held: {days} days | Suggested: {suggested_hold_period}"
- suggested_hold_period comes from analyze_trade's momentum.holding_period field, saved at trade creation
- Color: green if within period, amber if >75% elapsed, red if past suggested period
- Holding period string to days mapping: "short (2-4 weeks)" -> 28, "medium (1-3 months)" -> 90, "long (3-6 months)" -> 180
- "Days remaining" = mapped_days - (today - entry_date). Display countdown.

**Step 11: Daily P&L summary row**
- At top of Trade Builder tab, before ticker selection
- 4 metrics in a row: Total Unrealized P&L | Total Realized P&L | Best Today | Worst Today
- "Best/Worst Today" = open position with highest/lowest unrealized P&L percentage
- Data source: compute_paper_stats (already computed)

### Phase 4: My Edge Tab (~45 min with CC)

**Step 12a: Signal Scorecard table**
- Group closed trades by signal_source
- Columns: Signal Source | Trades | Win Rate | Avg Return % | Edge vs SPY
- Edge vs SPY = avg trade return minus SPY return over same calendar holding period per trade
- SPY data optimization: fetch SPY once for full date range (min open date to max close date), then slice locally per trade. One API call, not N.
- Show "Need 5+ trades" when sample < 5

**Step 12b: Closed Trade Stats panel**
- Win rate, avg win %, avg loss %, profit factor (gross wins / gross losses)
- Expectancy per trade = (win_rate * avg_win) - (loss_rate * avg_loss)
- Avg hold time: winners vs losers (from timestamp to closed_at)
- Equity curve: plotly line chart of cumulative realized P&L in dollars by close date

**Step 12c: Behavioral Patterns panel**
- Conviction vs outcome scatter (x=conviction score, y=return %)
- Exit reason distribution pie chart
- Avg hold duration by exit reason
- Day-of-week patterns (trades opened on Monday vs Friday, outcome difference)

**Step 12d: Signal strength sparklines**
- For each signal_source with 3+ closed trades
- Sparkline = plotly sparkline of sequential trade returns (chronological)
- Source: trades table ordered by closed_at
- Note: sparse initially, grows richer as data accumulates
